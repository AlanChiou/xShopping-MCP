## How to Contribute

### 1. **Report Issues or Suggest Features**
- Use GitHub Issues to report bugs, request new features, or propose changes to the specification.
- For tool additions or major changes, please open an issue first for discussion.

### 2. **Submit Pull Requests**
- Fork the repository
- Create a new branch (`git checkout -b feature/my-change`)
- Make your changes
- Ensure documentation is updated (especially if modifying tools or authentication)
- Add or update examples in the `examples/` folder if relevant
- Commit with clear, descriptive messages
- Open a Pull Request with a detailed description

**Pull Request Guidelines**
- Keep changes focused and small when possible
- Follow existing style: clear Markdown, consistent JSON formatting
- Reference related issues (e.g., "Closes #12")
- Be respectful and constructive

### 3. **Improve Documentation**
- Fix typos, improve clarity, or add translations
- Enhance examples or add new ones
- Suggest better explanations for complex concepts

### 4. **Implement SAP**
- Build a SAP server for your e-commerce platform (Rakuten, Shopify, WooCommerce, custom store, etc.)
- Share your implementation publicly or privately for feedback
- Contribute reference code snippets or full open-source implementations

### 5. **Spread the Word**
- Share SAP on X, LinkedIn, developer forums, or internal company channels
- Write blog posts, demos, or videos showing SAP in action
- Encourage your company or partners to consider adoption
